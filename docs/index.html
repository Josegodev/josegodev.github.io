Resumen

Mi foco es construir sistemas IA operables, no solo chatbots: runtimes locales con FastAPI, RAG documental, proveedores LLM intercambiables, integracion Telegram y trazabilidad JSON por ejecucion.

El valor tecnico esta en controlar el flujo completo: entrada, contrato, retrieval, proveedor de modelo, respuesta, metricas, errores y evidencia. El modelo puede cambiar; el runtime mantiene las reglas.

Presentacion

Del espacio a la IA local

Soy ingeniero con experiencia en sistemas industriales y aeroespaciales de alta exigencia. He trabajado como ingeniero de fabricacion en entornos criticos vinculados a Airbus e Indra, donde la trazabilidad, la documentacion tecnica, la validacion de procesos y el control de errores no son opcionales.

Esa experiencia me ha dado una vision practica de las dificultades reales que aparecen al introducir IA en sistemas existentes: integracion con procesos, control de calidad, dependencia de proveedores, falta de trazabilidad, gestion de errores, coste operacional y resistencia a convertir prototipos en herramientas mantenibles.

Desde esa perspectiva desarrollo NUCLEO, una linea de trabajo centrada en construir backends IA controlados, trazables y reproducibles. La idea principal es que el modelo no sea el sistema completo, sino una pieza sustituible dentro de una arquitectura con contratos, politicas, observabilidad, recuperacion documental y evaluacion.

Como parte de esa investigacion mantengo un laboratorio local, LOCALES, donde pruebo modelos locales, RAG documental, FastAPI, Telegram, trazas JSON, metricas por ejecucion y separacion entre runtime, proveedor LLM y capa de aplicacion.

El objetivo no es crear otro chatbot, sino explorar como llevar la IA generativa hacia sistemas operables: medibles, depurables, auditables y adaptables a entornos reales.

Stack tecnico

Backend

Python, FastAPI, contratos HTTP, validacion de payloads, separacion por capas y servicios operables.

IA / LLM

RAG documental, proveedores Ollama/OpenAI, modelos locales cuando aplica, prompts versionables y politicas de respuesta.

Datos

JSON, trazas de ejecucion, metadatos de retrieval, chunking documental y salidas auditables.

Operacion

Trazas JSON/JSONL, medicion de latencia, tokens, errores, coste estimado y evals deterministas para detectar drift.

Frontend / Integraciones

HTML, CSS, interfaces web simples, bots de Telegram y APIs para conectar sistemas existentes.

Estado real

Maduro para explicar

Prototipo avanzado y backend local experimental con contratos, RAG, adapters de proveedor, Telegram, trazas JSON y evals.

Demo curada

Presentable para reclutadores si se usan ejemplos sinteticos y se separa claramente runtime, laboratorio y artefactos privados.

No deployment-ready

Faltan empaquetado reproducible, requirements raiz, Dockerfile minimo, parametrizacion completa y healthchecks ampliados.

No production-ready

No hay validacion publica de alta concurrencia, observabilidad centralizada, hardening productivo ni corpus RAG publicable.

Limitaciones actuales

  • Proyecto experimental.
  • Observabilidad local basada en JSON.
  • Sin alta concurrencia validada.
  • Cloud en fase de estudio.
  • RAG dependiente de calidad documental y estrategia de chunking.
  • El repositorio publico muestra una vista curada, no artefactos privados ni logs reales.
