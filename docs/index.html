Not just chatbots or AI demos. I build the control layer around language models: retrieval, context, prompts, model calls, validation, traces, metrics, errors, replayability and evaluation.

A harness is the execution layer that makes a local LLM usable as a software component. The model is only one part of the system; around it, the harness controls what context it receives, what evidence it uses, how each answer is traced, how failures are detected and how different models can be compared under the same conditions.

Architecture comparison

Controlled AI Runtime: Cloud, OpenClaw and NUCLEO

This comparison shows three different ways to structure AI systems. Cloud platforms prioritize managed services and fast deployment. OpenClaw represents a persistent agent-oriented approach with routing, plugins and background workflows. NUCLEO (LOCALES) focuses on controlled execution: the model is only one replaceable component inside a governed backend runtime.

Cloud

Cloud AI platforms are useful when speed, scalability and managed services matter most. The trade-off is stronger coupling to the provider stack, less internal control over orchestration, and higher dependency on external pricing, APIs and operational limits.

OpenClaw

OpenClaw represents a more agent-oriented architecture: persistent background processes, routing, plugins, session queues and multi-agent workflows. This is powerful for continuous automation, but it also increases operational complexity.

NUCLEO (LOCALES)

NUCLEO is focused on controlled AI execution. A request enters through FastAPI and moves through explicit runtime boundaries: AgentService, AgentRuntime, Planner, PolicyEngine, ToolRegistry, RAG evidence and controlled execution.

What NUCLEO solves

NUCLEO is not another chatbot. It solves the operational problem of making LLM execution observable, reproducible and debuggable. Each run can record which model was used, which prompt version was active, what evidence was retrieved, which tool was called, whether policy allowed it, how many tokens were consumed, how long it took and what failed.

  • The model proposes, but the runtime validates.
  • Tools are executed through explicit contracts.
  • RAG evidence is tracked with chunks, sources and retrieval status.
  • Dry runs allow validation without real execution.
  • Approval gates separate planning from action.
  • Traces make debugging and evaluation possible.
  • Local and hybrid model providers can be swapped without rewriting the runtime.

The objective is controlled AI execution: observable, reproducible, debuggable and independent from blind dependency on a single model provider.

Plain English

Technical terms, in plain English

LLM

A Large Language Model is a probabilistic system that generates likely text from context. It can be useful, but it can also produce fluent wrong answers.

RAG

Retrieval-Augmented Generation means retrieving relevant documents before asking the model to answer. The goal is to ground the answer in available evidence.

JSON contracts

JSON is used as a structured format for requests, responses, configuration and execution records. It makes system behavior easier to validate and debug.

Trace / trace_id

A trace is the execution record of one run: what was asked, which model was used, what evidence was retrieved, how many tokens were consumed, how long it took and what failed.

Hallucination

A hallucination is a confident answer not properly supported by evidence. The goal is not to pretend hallucinations disappear, but to make them easier to detect, reduce and evaluate.

Tokens

Tokens are small pieces of text processed by the model. Tracking input and output tokens helps estimate cost, latency and workload.

Runtime

The runtime is the controlled execution layer around the model. It handles requests, applies rules, routes providers, records traces and prevents the model from owning the whole system.

Provider adapter

A provider adapter is a small integration layer that lets the backend talk to different model providers, such as a local model server or a cloud API, without rewriting the whole application.
