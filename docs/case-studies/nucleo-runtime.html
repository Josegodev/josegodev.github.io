NUCLEO — Runtime backend para agentes IA controlados

Separacion entre modelo, runtime, contratos, politicas y herramientas para evitar acoplamientos dificiles de operar.

Problema

Muchos prototipos IA mezclan prompt, modelo, herramientas, reglas de negocio y salida final en una unica capa. Esto dificulta reproducir errores, aplicar politicas y entender que ocurrio en cada ejecucion.

Decision arquitectonica

Diseñar un runtime backend con API FastAPI, contratos de entrada y salida, politicas explicitas, seleccion de modelo desacoplada y trazas por ejecucion.

Flujo simplificado

Solicitud HTTP -> validacion -> politica -> seleccion de modelo -> herramientas opcionales -> respuesta -> traza JSON.

Que demuestra

Capacidad para convertir un prototipo IA en un backend inspeccionable, con puntos claros de control, metricas y errores analizables.

Stack

Python, FastAPI, JSON, Docker, Ollama / LM Studio como runtimes locales de modelo.

Estado actual

Experimental. Base funcional para iterar contratos, politicas, trazabilidad y ejecucion local.

Limitaciones

  • Sin validacion de alta concurrencia.
  • Persistencia de observabilidad basada en archivos JSON.
  • No sustituye un sistema de monitorizacion productivo.
