Proyectos NUCLEO RAG

Caso tecnico

Web + Telegram + observabilidad

Integracion de canales de uso con trazabilidad por ejecucion, control de errores y metricas utiles para operar.

Problema

Las integraciones IA suelen fallar cuando no existe visibilidad sobre origen de la solicitud, modelo usado, latencia, tokens, errores y estado de retrieval.

Decision arquitectonica

Unificar las entradas web y Telegram en un backend comun, registrar trazas JSON por ejecucion y mantener el canal separado de la logica de modelo.

Flujo simplificado

Usuario web o Telegram -> adaptador de canal -> API backend -> runtime IA -> respuesta -> traza con metricas y estado.

Que demuestra

Capacidad para integrar IA en canales reales sin perder control operativo, manteniendo trazabilidad y separacion entre interfaz, backend y modelo.

Stack

FastAPI, Telegram Bot API, HTML/CSS, JSON logs, Docker, modelos locales segun disponibilidad.

Estado actual

Experimental. Flujo preparado para pruebas locales e iteracion de metricas antes de endurecer despliegue.

Limitaciones

  • No incluye gestion avanzada de colas.
  • Sin SLOs formales ni panel externo de observabilidad.
  • El coste se estima segun metrica disponible y puede ser cero en modelos locales.
